# Moduł 0 — Setup i rytm pracy

**Czas trwania:** 1 tydzień
**Pozycja w kursie:** pierwszy moduł — bez niego nie zaczynaj kolejnych.

## Cel modułu

Ustawić powtarzalny tygodniowy rytm nauki, który przetrwa do końca planu (6–9 miesięcy), oraz przygotować infrastrukturę: repo nauki w git i dziennik decyzji. Ten moduł nie uczy architektury — buduje system, dzięki któremu nauka architektury w ogóle się wydarzy.

Plan Architect Track jest długi. Najczęstszą przyczyną porażki takich planów nie jest trudność materiału, tylko rozpad rytmu około trzeciego tygodnia. Moduł 0 jest po to, żeby ten rozpad przewidzieć i mu zapobiec — zanim zainwestujesz miesiące pracy.

## Rytm tygodniowy, który ustawiasz w tym module

- **4×/tydzień** — 2h głębokiej nauki/praktyki (rano, jako pierwszy "chunk" w DayChunks)
- **1×/tydzień** — pisanie posta na LinkedIn (60–90 min)
- **3–4×/tydzień** — 15 min komentowania na LinkedIn
- **1×/tydzień** — przegląd tygodnia: co działa, co wyciąć

## Definition of Done modułu

Moduł 0 jest zaliczony, gdy **wszystkie** poniższe są prawdą:

- [ ] W DayChunks istnieje powtarzalny szablon tygodnia z 4 blokami głębokiej nauki, blokiem na post LinkedIn i blokiem przeglądu tygodnia — i przeżył pierwszy realny tydzień (nie tylko istnieje na papierze).
- [ ] Repo nauki jest w git, ma ustaloną strukturę katalogów i co najmniej kilka commitów z prawdziwej pracy (nie tylko "initial commit").
- [ ] `progress.md` ma pierwszy wpis tygodniowy.
- [ ] `notes/decisions-log.md` istnieje i zawiera minimum 2 wpisy w formacie kontekst → decyzja → konsekwencje.
- [ ] Odbył się pierwszy cotygodniowy przegląd i jego wynik (co działa / co wyciąć) jest zapisany.

Jeśli pod koniec tygodnia rytm "prawie działa" — to wystarczy, żeby przejść dalej. Rytm będziesz kalibrować przez cały kurs; moduł 0 ma go tylko uruchomić i dać narzędzia do korygowania.
